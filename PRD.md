# OC-Review — PRD

> AI Native 面试助手 · 基于 LLM 能力驱动的求职面试产品

---

## 一、产品定位

**目标用户**：求职者（技术岗位为主）

**一句话描述**：一款 AI Native 的面试准备与复盘工具，通过对话式交互帮助求职者模拟面试、复盘表现、持续提升面试能力。

---

## 二、AI Native 理念

不只是"套了 AI 的 CRUD 应用"，而是：

1. **对话即界面** — 主界面就是聊天，所有功能通过对话触发
2. **上下文智能** — AI 记住你是谁、目标是什么、之前练过什么
3. **主动交互** — 不仅响应用户提问，AI 在适当时机主动建议（"你下周有 Google 面试，要练一下 System Design 吗？"）
4. **渐进式披露** — 不需要复杂的设置页面，AI 在对话中逐步了解用户

---

## 三、核心功能

按求职者使用流程划分：

### Phase 1: 基础设定（Onboarding）

通过 Slash Command 配置个人信息：

- 输入 `/settings` 调出配置弹窗，填写/编辑个人信息
- 首次进入时，命令提示区域会自动展示可用命令，引导用户发现 `/settings`

配置项：

| 字段 | 说明 | 必填 |
|------|------|:---:|
| 目标岗位 | 如「前端工程师」「后端工程师」 | ✓ |
| 技术栈 | 如 React、Node.js、Python 等 | - |
| 工作年限 | 如「3年」 | - |
| 目标公司 | 如 Google、字节跳动（可多选） | - |
| 简历 | PDF 上传，AI 解析提取关键信息 | - |

这些信息作为**长期上下文**，所有对话都基于此。

### Phase 2: 核心功能模块

| 功能 | 描述 | AI Native 程度 |
|------|------|:---:|
| **模拟面试** | 用户上传简历后，AI 扮演面试官，按岗位/公司风格进行模拟面试 | ⭐⭐⭐ |
| **面试复盘** | 用户粘贴面试记录/回忆，AI 分析表现、指出改进点 | ⭐⭐⭐ |
| **题目预测** | 基于 JD + 简历，生成可能的面试题 | ⭐⭐ |
| **答案优化** | 用户写的答案，AI 给出结构化反馈（STAR 法则等） | ⭐⭐ |
| **随堂练习** | 聊天中 AI 主动出题，用户实时作答并收到反馈 | ⭐⭐⭐ |

### Phase 3: 长期价值

- **历史记录** — 所有模拟面试和复盘可回顾
- **进步轨迹** — 追踪回答质量变化，识别薄弱环节
- **个性化推荐** — 基于薄弱点推荐练习方向

## 四、产品交互

### 页面布局

侧栏作为导航中枢，选中不同项时右侧内容区随之切换。

```
┌──────────┬──────────────────────────────────┐
│          │     [右侧内容区 — 当前聊天]        │
│  侧栏     │                                  │
│  (模式    │   聊天框输入 /mock 等命令         │
│   历史)   │   → 进入对应 Mode                │
│          │   → 对话内容自动归入对应分类        │
│  · 聊天   │                                  │
│  · mock  │                                  │
│  · review├──────────────────────────────────┤
│  · practice│  [输入框 + 命令提示]               │
│  · predict│                                  │
│  · optimize│                                  │
│  · settings│                                  │
└──────────┴──────────────────────────────────┘
```

- **侧栏**：固定导航，默认折叠，点击展开。命名与 Slash Command 一致：
  - **chat**：默认自由对话，所有未分类对话
  - **mock**：查看历史模拟面试记录
  - **review**：查看历史复盘记录
  - **practice**：查看历史随堂练习记录
  - **predict**：查看历史预测记录
  - **optimize**：查看历史答案优化记录
  - **settings**：用户画像、模型供应商 API Key、Token 用量
- **右侧内容区**：始终为当前聊天界面。点击侧栏项切换到对应分类的历史记录列表，点击某条记录后右侧加载该对话
- **Slash Command 进入 Mode**：聊天框中输入 `/mock` 等命令，当前对话切换为对应模式，对话内容自动归类到该模式下

### Slash Command 交互系统

所有核心功能可通过 `/` 斜杠命令在聊天框中快速触发。除 `/settings` 外，其余命令进入对应的 Mode；Slash Command 与侧栏导航相互独立，互不影响。

#### 命令列表

| 命令 | 描述 | 行为 |
|------|------|------|
| `/settings` | 打开配置 | 特殊：跳转至侧栏 settings tab，不进入 Mode |
| `/mock` | 开始模拟面试 | 聊天框内切换为面试官模式，按岗位/公司风格提问 |
| `/review` | 复盘面试 | 提示用户粘贴面试记录，AI 分析并给出改进建议 |
| `/practice` | 随堂练习 | AI 随机出一题，用户作答后获得结构化反馈 |
| `/predict` | 预测面试题 | 基于用户画像生成可能的面试题列表 |
| `/optimize` | 优化答案 | 用户输入答案草稿，AI 按 STAR 法则等优化 |

#### 交互流程

```
用户输入 "/"  →  弹出命令建议面板（浮在输入框上方）
                  ↓
          键盘 ↑↓ / 鼠标点击选择命令
                  ↓
        命令填入输入框，Enter 后进入对应对话模式
```

**Slash Command 仅作用于聊天框，不会改变侧栏选中状态或触发页面路由跳转。**

#### 命令提示

- 输入框下方始终展示可用命令提示
- 用户无需记忆命令，发现即用
- 支持快捷键（如 Ctrl+K）用于高级用户

## 六、数据持久化策略

出于用户隐私保护考量，以下数据**不存储到云端数据库**：

| 数据类别 | 说明 | 处理方式 |
|---------|------|---------|
| 模型供应商配置 | API Key、供应商选择 | 客户端 IndexedDB，仅本地留存 |
| 用户简历 | PDF 文件及解析内容 | 客户端 IndexedDB（支持 Blob 存储） |
| 聊天记录 | 用户与 AI 的对话历史 | 客户端 IndexedDB，与服务器仅做传输不做存储 |
| 用户画像 | 目标岗位、技术栈等 | 客户端 IndexedDB |

选用 IndexedDB 而非 localStorage 的原因：
- **大容量**：聊天记录会随时间增长，localStorage 5-10MB 上限不够
- **Blob 支持**：简历 PDF 是二进制文件，IndexedDB 原生支持
- **异步 API**：不阻塞主线程

**MVP 阶段**：所有动态数据保留在客户端，不引入 Neon DB 或其他云端存储。后续可根据用户反馈评估是否需要可选的云端同步能力。
